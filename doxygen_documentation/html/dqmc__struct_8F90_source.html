7  !
8  ! This module defines the data type for describing underline lattice
9  ! struture used in DQMC. Basically, it contains six arrays.
10  ! Suppose the lattice has N sites.
11  !
12  ! 1. Adjacenty (T) is an NxK array, where K is the
13  ! maximum number of adjacent sites (=max_adj). T(0,i) records
14  ! the number of adjacency of site i. T(1:T(0,i),i) records
15  ! the sites that are adjacent to site i. The order of
16  ! these adjacency information is important if the 'checkboard
17  ! method' is used to construct Green's function. See DQMC_Gfun
18  ! for more details.
19  !
20  ! 2. Distance (D) is an NxN symmetric matrix. D(i,j) records the
21  ! distance information between site i and site j. However, the
22  ! data stored in D is not 'real distance' in any mathematical
23  ! sense. It is just an index of a distance table, which classifies
24  ! all the possible distance in the lattice. In other words,
25  ! if two elements in D have the same id, they have the same
26  ! distance. The total number of possible distance is nClass.
27  !
28  ! 3. Frequency (F) is a length nClass vector, which stores the
29  ! counts for each unique distance. For example, F(i)=5 means
30  ! there are 5 elements in D whose distance class is indexed i.
31  !
32  ! 4. Phase (P) is also an NxN matrix, which is used in computing
33  ! some phiscal measurements and Green's function. See DQMC_Phy0
34  ! and DQMC_Phy2 for more details.
35  !
36  ! 5. Neighborhood (N) is a NxB array, where B is the maximum
37  ! number of neighbors (=max_nbr).
38  ! N(0,i) records the number of neighbors
39  ! of site i. N(1:N(0,i),i) is a list of sites neighboring to
40  ! site i. The order of neighbor should be consistant.
41  !
42  ! 6. Wave (W) is a BxB matrix where B is the maximum number of
43  ! neighbors. This function is used in computing pair measurements.
44  ! See DQMC_Phy2 for more details.
45  !
46  ! Data Type
47  ! =========
